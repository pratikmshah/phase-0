Why You Should Git It!

10/3/2015

In this post I am going to talk to you about how a piece of software that is not as “gooey” as you would expect but plays a big role when managing source code versions. It is also open source, meaning it is free to use! It is called version control. Version control is simply the process of tracking and managing file changes over time. But, why use version control for your source code? Well, have you ever had code you were working on and found out it didn’t quite work only to wish you could go back to a stable version? Or what about when each member of your team is working on several files and everyone has to consolidate their work back to the original how would you track that? What about if you wanted to know who wrote the code, what they changed and why they changed it? Implementing version control will help solve all of these issues and more.

Ok so now you understand version control so what is this genius piece of free software that will help track all of this? It is called Git. Developers use Git because of its simple and fast design. Although there are some graphical user interfaces (GUI) clients available, in its true form Git runs purely through command line interface or interpreter (CLI). The CLI is a way for you to navigate and manipulate your system’s files through text commands by terminal/console. Git allows you to check and progress changes to your files. It consist of three phases: the working phase, the staging phase, and the commit phase. Afterwards you are able to do a pull/merge request.

In the working phase you create a copy of the file on your local drive and then create a new “branch” (subdivision) to make your changes. Once you finish your editing you move to the staging phase where you add the file to be tracked or “staged” by Git. Next, your file moves to the commit phase where you finalize and “commit” (save) the changes. Last, you send your files back to the remote location, such as GitHub; more about that later. You then create a pull request for your code to be reviewed and approved to be merged with the original source.

GitHub is a great complementary source where people are now able to host their source files into repositories on the cloud. GitHub provides some great advantages when it comes to version control and collaboration. One reason is that your code is hosted remotely so if your computer dies you will be able to retrieve your files to the most recent stable version. Another reason is that it allows for your code to be “forked” (copied) and to be worked on by anyone allowing for growth and improvements on the project. In addition, it allows for code review and collaboration before the changes are “merged” (consolidated) with the original source file.

I hope you have taken a basic understanding on what the purpose of Git is and how it helps with version control. Also, what a supplementary tool GitHub is to Git. Remember, version control helps with who, what, why and recovery of files over time.
